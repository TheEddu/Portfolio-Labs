Introdução

O laboratório Pickle Rick do TryHackMe é um desafio divertido e didático, inspirado na famosa série Rick and Morty. O objetivo é obter três flags escondidas em diferentes níveis de acesso ao sistema, praticando técnicas essenciais de enumeração, exploração de aplicações web e escalonamento de privilégios em ambientes Linux.

Este relatório detalha cada etapa do processo, destacando boas práticas, raciocínio lógico e ferramentas utilizadas para alcançar o objetivo final. O lab é excelente para quem está começando a praticar pentest, pois exige atenção a detalhes, criatividade e domínio de comandos básicos do Linux.

Enumeração Inicial

O primeiro passo em qualquer pentest é identificar os serviços expostos pela máquina alvo. Para isso, utilizamos o nmap, uma das ferramentas mais tradicionais para varredura de portas e serviços: 

nmap --open -v -sS -Pn MACHINE_IP
  • -sS: varredura SYN (stealth scan, menos detectável)
  • -Pn: ignora o ping, assume que o host está up
  • --open: mostra apenas portas abertas
  • -v: modo verbose, exibe mais detalhes

O resultado mostra as portas 22 (SSH) e 80 (HTTP) abertas. O SSH pode ser útil em etapas avançadas, mas inicialmente focamos na porta 80, onde geralmente há uma aplicação web.

Acessando o endereço no navegador, encontramos um formulário de login (login.php). Uma das melhores práticas é sempre inspecionar o código-fonte da página (Ctrl+U), pois desenvolvedores podem deixar comentários ou informações sensíveis.

No código-fonte, encontramos o nome de usuário anotado como comentário. Outro passo essencial é verificar o arquivo robots.txt (http://MACHINE_IP/robots.txt), que pode conter diretivas para mecanismos de busca, mas muitas vezes revela arquivos ou strings sensíveis.

No robots.txt, encontramos uma sequência de caracteres que, ao ser testada como senha junto ao usuário encontrado, permite o acesso ao sistema.

Esse processo evidencia a importância de sempre analisar arquivos e comentários aparentemente "inofensivos" durante a fase de reconhecimento.

Acesso e Exploração do Sistema

Após o login, somos direcionados a um painel de comandos web. Esse tipo de funcionalidade é bastante perigosa em aplicações reais, pois pode permitir a execução arbitrária de comandos no servidor.

O primeiro comando a ser testado é o clássico ls, que lista os arquivos do diretório atual. Com isso, identificamos arquivos interessantes, incluindo um .txt com a primeira flag.

Para visualizar o conteúdo da flag, podemos acessar diretamente pela URL.

Para avançar, é importante explorar a estrutura de diretórios do sistema. Utilizamos: 

cd /home; ls -al; pwd
  • cd /home: acessa o diretório home
  • ls -al: lista todos os arquivos, inclusive ocultos, com detalhes
  • pwd: exibe o caminho atual

Descobrimos os usuários rick e ubuntu. O próximo passo é explorar o diretório do usuário Rick, pois geralmente é onde flags e arquivos sensíveis são armazenados: 

cd /home/rick; ls -al; pwd

Encontramos o arquivo second ingredients, que contém a segunda flag. Para ler seu conteúdo, usamos: 

less /home/rick/"second ingredients"

O comando less é útil para visualizar arquivos longos ou com espaços no nome. Alternativamente, cat também pode ser utilizado.

Flag Final - Root

Para obter a última flag, precisamos acessar o diretório /root/, que normalmente só é acessível por usuários com privilégios elevados (root). O painel de comandos permite o uso do sudo sem senha, o que é uma falha grave de configuração.

Listamos o conteúdo do diretório root: 

sudo ls /root/

O arquivo 3rd.txt está presente. Para visualizar seu conteúdo, usamos: 

sudo less /root/3rd.txt

O uso do sudo para acessar arquivos do root demonstra a importância de configurar corretamente as permissões de sudoers em sistemas Linux.

Com isso, todas as flags foram encontradas, demonstrando o fluxo completo de enumeração, exploração e escalonamento de privilégios.

Conclusão e Aprendizados

  • A importância da enumeração detalhada, desde o reconhecimento de portas até a análise de arquivos ocultos.
  • A análise de código-fonte e arquivos como robots.txt pode revelar informações cruciais.
  • Acesso a painéis restritos pode ser obtido com criatividade e atenção a detalhes.
  • A escalada de privilégios é fundamental para obter controle total do sistema e acessar arquivos protegidos.
  • O lab reforça a importância de boas práticas de desenvolvimento e configuração segura de sistemas.

O desafio Pickle Rick é uma excelente introdução à metodologia de pentest, proporcionando uma experiência prática e divertida para quem está começando na área de segurança ofensiva.
